# Implementation plans

Store approved design and implementation plans here as `.pi/plans/<bead-id>-<topic>.md`.

Use one location for each kind of future work:

- **Beads:** actionable tasks, bugs, decisions, dependencies, approvals, and status.
- **`docs/FUTURE_FEATURES.md`:** researched ideas that are not yet committed work.
- **`.pi/plans/`:** execution plans for an approved Bead.
- **`README.md`:** current capabilities and a short public status summary, not a backlog.

A future-feature idea should move into Beads when someone chooses it for execution. Create its plan here only when the work needs a durable multi-step design or implementation sequence.
